# AI_CONTEXT.md

Главный документ-контекст проекта WhiteListCheck для работы с ИИ-ассистентами.
Прочитав его, ИИ должен быть готов продолжить работу без чтения истории чатов.

## Назначение проекта

Android-приложение **«Белый список?»** помогает пользователю в РФ понять,
что происходит с его интернетом: включён ли в регионе режим «белого списка»
мобильного интернета (перечень доступных сайтов, который ведёт Минцифры),
пропал ли интернет целиком, работает ли VPN — или всё в норме.

Проект создан пользователем-новичком (Дмитрий / Dmitry Starosta,
GitHub: dmitrystarosta) полностью с помощью ИИ. Это его первое приложение.

## Краткое описание приложения

По нажатию кнопки «Проверить» приложение параллельно опрашивает три группы
сайтов лёгкими HTTPS HEAD-запросами (к favicon.ico, таймаут 4 с):

| Группа | Смысл | Состав (встроенный список) |
|---|---|---|
| A — белый список | эталон доступности | Яндекс, ВКонтакте, Госуслуги, Mail.ru |
| B — обычный рунет | сайты вне списка | Habr, 4PDA, Drive2, Banki.ru |
| C — заблокированные в РФ | контрольная группа | Instagram*, X (Twitter), Rutracker |

Вердикт по большинству в группе:
- A работает, B нет → «включён белый список»
- A и B работают, C нет → «всё в норме»
- Работает всё, включая C → «включён VPN или вы вне РФ»
- Не работает ничего → «интернет недоступен вообще»

## Основные функции

- Ручная проверка одной кнопкой со «светофором» по каждому сайту
- Фоновая проверка (WorkManager, ~15 мин) с уведомлением при смене режима
- Определение типа сети (мобильный / Wi-Fi / VPN / нет сети) с пояснениями;
  предупреждение, что Wi-Fi от 3G/4G-роутера тоже попадает под белые списки
- Названия сайтов кликабельны — открывают сайт в браузере
- Ошибки переведены на русский: «адрес не найден (DNS)», «нет ответа (таймаут)»,
  «соединение сброшено», «ошибка шифрования (TLS)» + сноска-объяснение
- Подвал: версия приложения (кликабельная ссылка на страницу релизов) и
  «© 2026, Dmitry Starosta»
- Заготовка удалённого конфига списков (ProbeConfig.REMOTE_CONFIG_URL, пока пустая)

## Архитектура

Однофайловое приложение: весь код в MainActivity.kt (~450 строк).
Внутри файла: data-классы → ProbeConfig (списки) → Scanner (сетевые проверки,
вердикт) → CheckWorker + schedule/cancelBackground (фон) → MainActivity →
Composable-функции UI (App, VerdictCard, GroupHeader, ProbeRow, Footnote,
AppFooter). Состояние — ScanState через remember/mutableStateOf.
Настройки (bg_enabled, last_verdict) — в SharedPreferences "netstatus".

- HttpURLConnection (без сторонних HTTP-библиотек), kotlinx-coroutines

## Важные ограничения (категорически нельзя менять)

1. **applicationId и пакет `ru.netstatus.app`** — это идентификатор приложения
   в Android и будущий ID в RuStore. Смена = «другое приложение» для системы.
   Имя пакета НЕ совпадает с названием репозитория — это нормально.
2. **Тема в манифесте только `@android:style/Theme.Material.Light.NoActionBar`.**
   Библиотечные темы (Theme.Material3.*) НЕ подключены — их использование
   ломает сборку ошибкой «resource style/... not found» (наступали дважды).
3. **Никогда не коммитить в репозиторий keystore.properties и файлы ключей
   (*.jks)** — репозиторий публичный. Первый ключ был скомпрометирован именно
   так и перевыпущен. Пароли живут только в GitHub Secrets.
4. **Имена артефактов в build.yml** (name/path: WhiteListCheck / WhiteListCheck.apk)
   должны совпадать буква в букву в шагах Rename и Upload.
5. **gradle.properties с `android.useAndroidX=true` обязателен** — без него
   сборка падает.
6. **Иконка** — VectorDrawable ic_launcher.xml («чебурнет»: коричневый глобус
   с ушами, фон #4E342E, линии #FFCCBC). Прямые отсылки к Чебурашке
   (персонажу) сознательно избегаются из-за авторских прав Союзмультфильма.
7. **Сноска про Instagram** («принадлежит Meta, признанной экстремистской и
   запрещённой в РФ») — юридически обязательная формулировка, не удалять.
8. Версия приложения задаётся в app/build.gradle.kts (versionCode/versionName)
   и автоматически отображается в подвале UI — при релизе повышать оба.

## Как нужно работать с проектом

- Пользователь — новичок: НЕ пользуется git локально. Все правки — через
  веб-интерфейс GitHub: открыть файл → карандаш (Edit) → Ctrl+A → Delete →
  вставить новое содержимое → Commit changes.
- Поэтому ИИ должен выдавать **файлы целиком на замену** (или точечные правки
  «найди строку X — замени на Y»), с пошаговыми инструкциями на русском.
- Сборка запускается автоматически на каждый commit (это норма; промежуточные
  красные сборки при мультифайловых правках — ожидаемы; важна последняя).
- Готовый APK: вкладка Actions → верхняя зелёная сборка → Artifacts.
- Релизы оформляются вручную: Releases → Draft a new release → тег vX.Y →
  приложить APK → галка «Set as the latest release».
- Тестовое устройство: Xiaomi 13T Pro (MIUI/HyperOS).
- Язык общения и всех текстов приложения — русский.

## Известные особенности

- APK до перехода на release-подпись вызывал предупреждение Google Play
  Protect «App blocked» (debug-подпись) — устанавливается через
  More details → Install anyway. Проверка Xiaomi проходит чисто.
- MIUI агрессивно убивает фон: для работы фоновой проверки нужно вручную
  включить Автозапуск и снять ограничения батареи. Интервал 15 мин —
  минимум Android, система может растягивать.
- Один из старых jpg-скриншотов был усечён — при обработке изображений
  использовать PIL с LOAD_TRUNCATED_IMAGES.
- GitHub-редиректы со старого имени репозитория NetStatus работают.
- Android TV не поддерживается (нет leanback-манифеста, фокуса для пульта).
