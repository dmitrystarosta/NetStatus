# Размещение сайта belyjspisok.ru

## Рекомендация
Для этого лендинга оптимален **GitHub Pages + ваш домен belyjspisok.ru**. Сайт статический, поэтому отдельный платный хостинг не даёт заметных преимуществ. GitHub Pages бесплатен, автоматически выпускает HTTPS-сертификат и удобно обновляется вместе с репозиторием.

## Вариант 1: GitHub Pages
1. Создайте отдельный публичный репозиторий, например `belyjspisok-site`.
2. Загрузите в корень все файлы из этой папки.
3. В репозитории откройте **Settings → Pages**.
4. В разделе Build and deployment выберите **Deploy from a branch**, ветку `main`, папку `/ (root)`.
5. В поле Custom domain укажите `belyjspisok.ru` и сохраните.
6. У регистратора домена добавьте DNS-записи:
   - `A` для корня `@`: `185.199.108.153`
   - `A` для корня `@`: `185.199.109.153`
   - `A` для корня `@`: `185.199.110.153`
   - `A` для корня `@`: `185.199.111.153`
   - `CNAME` для `www`: `dmitrystarosta.github.io.`
7. После применения DNS включите **Enforce HTTPS**.
8. Добавьте в корень файл `CNAME` с одной строкой: `belyjspisok.ru` (GitHub обычно создаёт его сам).

## Вариант 2: обычный хостинг
Подойдёт любой тариф со статическими файлами и HTTPS. Загрузите содержимое папки в `public_html`, `www` или корень сайта через SFTP/FTP. Убедитесь, что `index.html`, `robots.txt` и `sitemap.xml` доступны из корня домена.

Платный хостинг оправдан, если позже появятся серверные функции: собственная база данных, API, личный кабинет, динамическое обновление списков или серверная аналитика.

## Яндекс.Метрика
В `index.html` найдите комментарий `ВСТАВЬТЕ КОД ЯНДЕКС.МЕТРИКИ` и вставьте код счётчика сразу после него. Для более чистой статистики настройте цели на клики по двум кнопкам скачивания.

## После публикации
- Добавьте сайт в Яндекс Вебмастер и Google Search Console.
- Подтвердите домен и отправьте `https://belyjspisok.ru/sitemap.xml`.
- Проверьте заголовок, описание и изображение через сервисы предпросмотра Open Graph.
- Не меняйте URL главной страницы без необходимости.
- При выходе важных версий обновляйте текст и скриншоты, но оставляйте ссылки на `releases/latest` и RuStore постоянными.
