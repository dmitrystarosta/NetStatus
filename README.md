# «Белый список?» — детектор режима ограничений мобильного интернета

Android-приложение (Kotlin + Jetpack Compose), которое одним нажатием
определяет, действует ли сейчас режим «белого списка».

## Как работает
Три группы проверок (лёгкие HTTPS HEAD-запросы к favicon, таймаут 4 с,
все параллельно):

| Группа | Что проверяет | Пример |
|---|---|---|
| A | Сайты из белого списка — эталон доступности | Яндекс, ВК, Госуслуги |
| B | Обычный рунет вне списка | Habr, 4PDA, Drive2 |
| C | Заблокированные ресурсы — контроль | Instagram*, X, Rutracker |

Вердикт по большинству в группе:
- A не работает и B не работает → интернета нет вообще
- A работает, B не работает → **белый список включён**
- A и B работают → всё в норме

Также приложение показывает тип сети и предупреждает, если вы на Wi-Fi
(там белые списки обычно не действуют, результат нерепрезентативен).

## Обновляемый список
В `ProbeConfig.REMOTE_CONFIG_URL` можно указать URL JSON-файла вида:
```json
{
  "a": [{"name": "Яндекс", "url": "https://ya.ru/favicon.ico"}],
  "b": [{"name": "Habr", "url": "https://habr.com/favicon.ico"}],
  "c": [{"name": "X", "url": "https://x.com/favicon.ico"}]
}
```
Важно: размещайте конфиг на хостинге, который сам входит в белый список
(объектное хранилище VK Cloud, Яндекса и т.п.), иначе в момент
ограничений обновление не скачается. Встроенный список остаётся запасным.

## Сборка
1. Откройте папку проекта в Android Studio (Hedgehog или новее).
2. Дождитесь синхронизации Gradle.
3. Run → на устройство или эмулятор. Либо `./gradlew assembleDebug`.
